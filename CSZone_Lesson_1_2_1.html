1.2.1 — Primary Storage (RAM & ROM)
Component 1: Computer Systems  ·  Free lesson  ·  ~20 min

What is Primary Storage?

Primary storage (also called main memory) holds data and instructions that the CPU is currently using. It is directly accessible by the processor and much faster than secondary storage.

There are three types of primary storage you need to know: RAM, ROM, and Cache memory.

RAM — Random Access Memory

Key Facts about RAM

  • Volatile — loses all data when the power is switched off
  • Temporarily stores currently running programs, the OS, and active data
  • Can be both read from and written to
  • Directly accessible by the CPU at high speed
  • Typical sizes: 4GB–64GB in modern computers
  • The more RAM installed, the more programs can run simultaneously without slowing down

Think of RAM like your desk — it holds everything you're currently working on. When you close the computer, everything on the desk is cleared.

Types of RAM

DRAM (Dynamic RAM)

The most common type used as main memory. Must be constantly refreshed to retain data. Slower but cheaper and higher capacity.

SRAM (Static RAM)

Faster and more expensive than DRAM. Does not need refreshing. Used for cache memory.

ROM — Read-Only Memory

Key Facts about ROM

  • Non-volatile — retains data when the power is off
  • Stores the computer's startup/boot instructions (BIOS or UEFI firmware)
  • Cannot normally be written to or modified
  • Much smaller capacity than RAM (typically just a few MB)
  • Read by the CPU immediately when the computer is switched on

Think of ROM like a printed instruction manual — the content is fixed and doesn't change. When you turn the computer on, the CPU reads ROM first to start up.

BIOS (Basic Input/Output System) — stored in ROM. It initialises hardware components and loads the operating system from secondary storage into RAM at startup.

Cache Memory

Key Facts about Cache

  • Extremely fast, small memory located between the CPU and RAM
  • Stores copies of frequently used instructions and data
  • Reduces the number of times the CPU must wait for data from slower RAM
  • Three levels: L1 (fastest, smallest, inside CPU core), L2 (medium speed/size), L3 (largest, shared between cores)
  • Made from faster (and more expensive) SRAM

Cache Hits and Misses

Cache Hit ✓

The CPU looks for data and finds it in the cache. Access is very fast. This is the ideal scenario.

Cache Miss ✗

The CPU looks for data but it is NOT in the cache. The CPU must fetch it from RAM (slower), causing a delay.

A higher cache hit rate means faster overall performance. Increasing cache size can increase hit rate but cache is expensive, so there is a practical limit.

Why Does RAM Size Matter?

When RAM is full, the OS may use virtual memory — a section of the hard drive used as an overflow. This is much slower than real RAM and can make the computer sluggish.

  • More RAM → more programs can run at the same time without slowdown
  • More RAM → less use of virtual memory → faster performance
  • RAM speed (measured in MHz) also affects performance

Common Mistakes

  • Confusing RAM (temporary) with hard drive storage (permanent)
  • Saying ROM stores programs — it stores only boot instructions (firmware)
  • Saying cache is the same as RAM — cache is faster and much smaller
  • Confusing L1/L3 — L1 is fastest and smallest; L3 is slowest and largest
