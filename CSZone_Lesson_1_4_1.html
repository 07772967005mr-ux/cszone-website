1.4.1 — Network Threats
Component 1: Computer Systems  ·  ~20 min  ·  Free lesson

Forms of Attack

Networks and systems face a range of threats. Understanding these is crucial for both the exam and real-world security.

Malware

  • Virus — self-replicating code that attaches to files; spreads when infected files are shared
  • Worm — self-replicating; spreads independently across networks without needing to attach to a file
  • Trojan — disguised as legitimate software; doesn't self-replicate but opens backdoors
  • Ransomware — encrypts victim's files, demands payment for decryption key
  • Spyware — monitors user activity, steals personal/financial data without consent

Social Engineering

  • Phishing — fraudulent emails/messages that trick users into revealing login credentials or personal data
  • Vishing — voice phishing (phone calls); attacker poses as trusted authority (bank, IT support)
  • Smishing — SMS phishing; fraudulent text messages
  • Shoulder surfing — physically observing someone entering passwords/PINs
  • Blagging — inventing a scenario to obtain information or access (also called pretexting)

Technical Attacks

  • Brute force attack — systematically tries every possible password combination until correct one found
  • Denial of Service (DoS) — floods a server with requests so legitimate users cannot access it
  • Distributed DoS (DDoS) — DoS from many compromised computers (botnet) simultaneously
  • Man-in-the-Middle (MitM) — attacker intercepts communication between two parties without their knowledge
  • SQL injection — malicious SQL code entered into a form field to manipulate a database

Common Mistakes

  • Confusing virus and worm — a virus NEEDS a host file to spread; a worm spreads independently
  • Forgetting DDoS is distributed (multiple machines) vs DoS (single machine)
  • Thinking phishing is only email — also includes text (smishing) and voice (vishing)
  • Not explaining HOW brute force works — must say "tries all combinations"
