2.1.3b — Binary Search
Component 2: Programming  ·  ~20 min

What is a Binary Search?

A binary search finds a target in a sorted list by repeatedly halving the search space. It compares the target with the middle element and eliminates the half that cannot contain the target.

Critical requirement: The list MUST be sorted before binary search can be applied. This is the most commonly tested constraint.

How Binary Search Works

Step-by-step:

  1. Set low = 0, high = length − 1
  2. Calculate mid = (low + high) DIV 2 (integer division)
  3. If list[mid] == target → return mid (found)
  4. If list[mid] < target → set low = mid + 1 (search right half)
  5. If list[mid] > target → set high = mid − 1 (search left half)
  6. Repeat from step 2 until found or low > high (not found)

OCR Pseudocode:

function binarySearch(list, target) low = 0 high = len(list) - 1 while low <= high mid = (low + high) DIV 2 if list[mid] == target then return mid elseif list[mid] < target then low = mid + 1 else high = mid - 1 endif endwhile return -1 endfunction

Example trace — searching for 7 in [1, 3, 5, 7, 9, 11, 13]:

low=0, high=6 → mid=3, list[3]=7 → 7==7 → return 3 (found in 1 comparison!)

Searching for 11:
Pass 1: low=0, high=6 → mid=3, list[3]=7; 11>7 → low=4
Pass 2: low=4, high=6 → mid=5, list[5]=11; 11==11 → return 5. Found in 2 comparisons!

Comparison: Binary vs Linear Search

Binary SearchLinear Search
Sorted list required?YesNo
Time complexityO(log n)O(n)
Speed (large lists)Much fasterSlower
ComplexityMore complex to writeSimple
O(log n): With 1,024 items, binary search needs at most 10 comparisons (log₂1024 = 10). Linear search needs up to 1,024.

Common Mistakes

  • Forgetting binary search REQUIRES a sorted list
  • Using normal division instead of integer division (DIV) for mid — mid must be a whole index
  • Saying binary search is "always better" — for unsorted lists, you must sort first (costly) or use linear search
  • Getting low/high updates wrong: target > mid → low = mid+1; target < mid → high = mid-1
