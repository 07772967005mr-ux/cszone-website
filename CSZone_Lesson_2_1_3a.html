2.1.3a — Linear Search
Component 2: Programming  ·  ~18 min

What is a Linear Search?

A linear search (also called sequential search) checks each item in a list one by one, from the start, until the target is found or the end of the list is reached.

Key fact: Works on UNSORTED data. No requirement for the list to be in order.

How Linear Search Works

Step-by-step:

  1. Start at the first element (index 0)
  2. Compare the current element with the target
  3. If they match → return the index (item found)
  4. If not → move to the next element
  5. Repeat until found or end of list reached
  6. If end reached without finding → return -1 (not found)

OCR Pseudocode:

function linearSearch(list, target) for i = 0 to len(list) - 1 if list[i] == target then return i endif next i return -1 endfunction

Example trace — searching for 7 in [3, 1, 7, 9, 2]:

i=0: list[0]=3, 3≠7 → continue
i=1: list[1]=1, 1≠7 → continue
i=2: list[2]=7, 7==7 → return 2 (found at index 2)

Advantages and Disadvantages

AdvantagesDisadvantages
Works on unsorted listsSlow for large lists — checks every item
Simple to understand and implementInefficient: O(n) — worst case checks all n items
Works on any data typeNot suitable for very large datasets
Complexity: Linear search is O(n) — time taken grows linearly with list size. Worst case: n comparisons (item at end or not present).

Common Mistakes

  • Saying linear search requires sorted data — it DOES NOT
  • Saying it always checks every item — it stops as soon as the item is found
  • Forgetting to handle the 'not found' case — return -1
