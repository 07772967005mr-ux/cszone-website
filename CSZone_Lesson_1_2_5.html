1.2.5 — Compression
Component 1: Computer Systems  ·  ~20 min

Why Compress Files?

Files can be very large — images, audio, and video especially. Compression reduces file size, which has several benefits:

  • Faster to upload/download over the internet
  • Less storage space needed
  • Faster to transmit/stream
  • Cheaper to store (cloud storage costs)

There are two main types of compression: lossless and lossy.

Lossless Compression

Lossless compression reduces file size without losing any data. The original file can be perfectly reconstructed from the compressed version.

How it works — Run-Length Encoding (RLE)

RLE is a simple lossless technique. Instead of storing every pixel/value, it stores a value and the number of times it repeats.

Original: AAAABBBBBCCDDDDDD
RLE: 4A 5B 2C 6D

18 characters → 8 values (more efficient)

Dictionary Encoding

Another lossless technique. Frequently occurring patterns are replaced by short codes stored in a dictionary. The decompressor uses the dictionary to restore the original.

Lossless file formats:

  • PNG — lossless image format
  • GIF — lossless (limited to 256 colours)
  • ZIP — lossless file archiving
  • FLAC — lossless audio
Lossless = No data is lost. File can be perfectly restored to original. Used when accuracy is critical (e.g. text files, medical images, executable programs).

Lossy Compression

Lossy compression achieves smaller file sizes by permanently removing some data. The original cannot be perfectly restored. The removed data is chosen to be the least noticeable to humans.

How it works — examples:

  • Images: remove fine detail and similar nearby colours that the eye barely notices. Store averaged/approximated colour blocks.
  • Audio: remove frequencies outside human hearing range (<20 Hz, >20,000 Hz) and quiet sounds masked by louder ones (perceptual coding).

Lossy file formats:

  • JPEG — lossy image format. Very high compression ratios possible.
  • MP3, AAC — lossy audio. Used for music streaming.
  • MP4 / H.264 — lossy video.
Lossy = Data is permanently removed. Cannot restore original. Smaller file size. Used for media where small quality loss is acceptable.

Lossless vs Lossy Comparison

FeatureLosslessLossy
Data lost?NoYes (permanently)
Can restore original?Yes, exactlyNo
Compression ratioModerateHigh (much smaller files)
Quality after decompressionIdentical to originalSlightly or noticeably degraded
Best used forText, programs, medical imagesPhotos, music, video streaming
ExamplesPNG, ZIP, FLACJPEG, MP3, MP4

Common Mistakes

  • Saying lossless removes some data — it removes none
  • Saying you can restore the original from lossy compression — you cannot
  • Confusing JPEG (lossy) with PNG (lossless)
  • Saying compression always reduces quality — lossless does not
  • Forgetting that lossy compression is irreversible
  • Not naming a specific technique (e.g. RLE, dictionary encoding) when asked how lossless works
