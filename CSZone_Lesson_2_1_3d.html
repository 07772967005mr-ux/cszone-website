2.1.3d — Merge Sort
Component 2: Programming  ·  ~22 min

What is Merge Sort?

Merge sort is a divide and conquer sorting algorithm. It works by repeatedly splitting a list in half until each sub-list has one element (which is trivially sorted), then merging those sub-lists back together in sorted order.

Key fact: Merge sort is O(n log n) — much more efficient than bubble sort's O(n²) for large datasets. It always takes O(n log n) regardless of input order.

Two Phases of Merge Sort

Phase 1 — Split (Divide):

  1. Find the midpoint of the list
  2. Split into left half and right half
  3. Recursively split each half
  4. Stop when each sub-list has only 1 element (a single element is always sorted)

Phase 2 — Merge (Conquer):

  1. Compare the first elements of two sorted sub-lists
  2. Take the smaller element and add it to the merged list
  3. Repeat until one sub-list is empty
  4. Append remaining elements from the non-empty sub-list
  5. The merged result is a sorted list

Full example — [5, 2, 8, 1, 4, 7]:

Split:
[5,2,8,1,4,7] → [5,2,8] and [1,4,7]
[5,2,8] → [5,2] and [8]  ·  [1,4,7] → [1,4] and [7]
[5,2] → [5] and [2]  ·  [1,4] → [1] and [4]
Merge:
[5] + [2] → [2,5]  ·  [1] + [4] → [1,4]
[2,5] + [8] → [2,5,8]  ·  [1,4] + [7] → [1,4,7]
[2,5,8] + [1,4,7] → [1,2,4,5,7,8] ✓

The Merge Step in detail — merging [2,5,8] and [1,4,7]:

Compare 2 vs 1: take 1 → merged=[1]
Compare 2 vs 4: take 2 → merged=[1,2]
Compare 5 vs 4: take 4 → merged=[1,2,4]
Compare 5 vs 7: take 5 → merged=[1,2,4,5]
Compare 8 vs 7: take 7 → merged=[1,2,4,5,7]
Left list empty, append remaining [8] → merged=[1,2,4,5,7,8]

Comparison: Merge Sort vs Bubble Sort

Merge SortBubble Sort
Time complexityO(n log n)O(n²)
Efficiency (large lists)Much fasterMuch slower
Memory usageExtra memory for sub-listsIn-place (minimal extra)
Complexity to codeMore complexSimple

Common Mistakes

  • Confusing the split and merge phases — splitting produces individual elements; merging recombines in order
  • Saying merge sort is O(n²) — it is O(n log n)
  • Forgetting that merge sort requires extra memory for sub-lists (not in-place)
  • Not appending the remaining elements after one sub-list is exhausted in the merge step
