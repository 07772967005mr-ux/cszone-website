1.1.2 CPU Performance Factors

Subtopic 2 of 3 · Component 1: Computer Systems

What Affects CPU Performance?

The speed at which a CPU can execute instructions depends on several key factors. For OCR J277, you need to know three: clock speed, number of cores, and cache size. Understanding each factor — and how increasing it improves performance — is essential.

1. Clock Speed

The clock speed (also called clock rate) is the number of cycles a CPU can complete per second. It is measured in hertz (Hz). Modern CPUs operate at billions of cycles per second — measured in gigahertz (GHz).

  • 1 GHz = 1,000,000,000 (one billion) cycles per second
  • Each clock cycle, the CPU can fetch, decode, or execute part of an instruction
  • A CPU running at 3.5 GHz can perform 3.5 billion cycles every second

Effect: A higher clock speed means more instructions can be processed per second, making the CPU faster. However, a higher clock speed also generates more heat and uses more power — this is why laptops and phones run at lower clock speeds than desktop PCs.

💡 Analogy Think of clock speed like a metronome. A faster metronome means the CPU ticks more often — more work done each second. Doubling the clock speed (roughly) doubles performance, all else being equal.
⚠️ Examiner Watch Always give the unit when discussing clock speed. "3 GHz" is better than just "3". Also, be precise: clock speed tells you how many cycles per second, NOT how many instructions per second (one instruction may take multiple cycles).

2. Number of Cores

A core is an independent processing unit within the CPU chip. Traditional CPUs had just one core. Modern CPUs have multiple cores — commonly 4, 8, or even 16 or more.

  • Dual-core: 2 cores
  • Quad-core: 4 cores
  • Octa-core: 8 cores

Effect: Multiple cores allow the CPU to work on several tasks simultaneously (in parallel). For example, a quad-core CPU could run your browser, your antivirus scan, your music app, and your OS processes all at the same time — each on a different core.

However, additional cores only improve performance if:

  1. The software is designed to use multiple cores (called multi-threaded software)
  2. There are actually multiple tasks that can run at the same time
⚠️ Common Mistake Students often say "double the cores = double the speed." This is NOT always true. If you're running a single task that can't be split across cores, extra cores won't help at all. The software must support parallelism.

3. Cache Size

The cache is a small, very fast memory that sits inside (or very close to) the CPU chip. It stores frequently used instructions and data so the CPU can access them quickly without going to main memory (RAM), which is much slower.

Cache operates at different levels:

LevelSizeSpeedLocation
L1 Cache32–512 KBFastestOn the CPU core itself
L2 Cache256 KB – 4 MBFastOn the CPU chip, shared between cores
L3 Cache4 MB – 64 MBSlower than L1/L2On the CPU chip, shared between all cores

Effect: A larger cache means more data can be stored close to the CPU, reducing the time wasted waiting for RAM. This is called reducing cache misses — when the CPU needs something not in cache, it must fetch it from RAM, causing a delay.

💡 Analogy Imagine a chef (the CPU) working in a kitchen. The cache is their work surface — the bigger it is, the more ingredients they can keep within arm's reach. RAM is the fridge further away. Every time they have to walk to the fridge (cache miss), they slow down.

Summary: How Each Factor Affects Performance

FactorHow it worksIncreasing it...
Clock speedCycles per secondMore instructions processed per second
Number of coresIndependent processorsMore tasks can run simultaneously
Cache sizeFast on-chip memoryFewer cache misses, less waiting for RAM
⚠️ Common Mistakes to Avoid
  • ❌ "More cores always makes it faster" — only if software supports multi-threading
  • ❌ "Cache is the same as RAM" — cache is faster, smaller, and on the CPU chip
  • ❌ "Higher GHz = more instructions per instruction" — GHz = cycles per second, not instructions
  • ❌ Forgetting that these factors can all improve performance together
