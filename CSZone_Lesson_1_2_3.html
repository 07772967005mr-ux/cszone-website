The Bit — The Smallest Unit

All digital data is stored and processed as binary — sequences of 0s and 1s. The smallest unit of data is a bit (binary digit). A single bit can hold either 0 or 1.

The Nibble and Byte

  • A nibble = 4 bits
  • A byte = 8 bits

A byte is the fundamental unit used to measure file sizes and storage capacity. One byte can represent 2⁸ = 256 different values (0–255).

Exam tip: OCR J277 uses the binary prefix system (1 KB = 1,024 bytes, not 1,000). Always use 1,024 in calculations unless told otherwise. A common calculation: "Convert 5 GB to MB: 5 × 1,024 = 5,120 MB."

Key Conversion Rules

  • ×1,024 when converting down (e.g. GB → MB: multiply by 1,024)
  • ÷1,024 when converting up (e.g. MB → GB: divide by 1,024)
  • 1 KB = 1,024 B; 1 MB = 1,024 KB; 1 GB = 1,024 MB; 1 TB = 1,024 GB

Why Binary Uses Powers of 2

Computers use binary (base 2), so storage units are based on powers of 2:

  • 2¹⁰ = 1,024 ≈ 1,000 → this is why 1 KB ≈ 1,000 bytes but exactly 1,024 bytes
  • 2²⁰ = 1,048,576 ≈ 1 million → 1 MB
  • 2³⁰ ≈ 1 billion → 1 GB
  • 2⁴⁰ ≈ 1 trillion → 1 TB

⚠️ Common Mistakes
  • Using 1,000 instead of 1,024 — always use 1,024 in OCR J277 calculations
  • Confusing bits (b) and bytes (B) — 1 byte = 8 bits; a lowercase b usually means bits
  • Forgetting the nibble is 4 bits (half a byte)
  • Going the wrong direction in conversions — to go to a bigger unit, divide; to go to smaller, multiply
  • Forgetting the order: Bit → Nibble → Byte → KB → MB → GB → TB → PB
