1.2.3 — Units of Storage
Component 1: Computer Systems  ·  ~20 min

Bits and Bytes

All data in a computer is stored and processed as binary — sequences of 0s and 1s. The smallest unit of data is a bit.

UnitAbbreviationSize
BitbA single binary digit (0 or 1)
Nibble—4 bits
ByteB8 bits
KilobyteKB1,024 bytes
MegabyteMB1,024 KB = 1,048,576 bytes
GigabyteGB1,024 MB
TerabyteTB1,024 GB
PetabytePB1,024 TB
Remember: Each unit is 1,024 times (2¹⁰) the one below it, not 1,000. This is because computers work in binary (base 2).

Converting Between Units

To convert between units, multiply or divide by 1,024:

To go UP (smaller → larger units): DIVIDE by 1,024
To go DOWN (larger → smaller units): MULTIPLY by 1,024

Example: 5 GB = 5 × 1,024 = 5,120 MB
Example: 2,048 KB = 2,048 ÷ 1,024 = 2 MB

Worked Examples

  • How many bytes in 3 KB? → 3 × 1,024 = 3,072 bytes
  • How many KB in 4 MB? → 4 × 1,024 = 4,096 KB
  • Convert 512 MB to GB → 512 ÷ 1,024 = 0.5 GB
  • How many bits in 1 byte? → 8 bits
  • How many bits in 1 KB? → 1,024 × 8 = 8,192 bits

Calculating File Sizes

Image File Size

An image is made of pixels. Each pixel requires a certain number of bits (the colour depth) to store its colour.

Image file size (bits) = Width (pixels) × Height (pixels) × Colour depth (bits per pixel)
Divide by 8 to get bytes, then 1,024 for KB, etc.

Worked Example — Image

An image is 800 × 600 pixels with a colour depth of 24 bits. Calculate the file size in MB.

Size = 800 × 600 × 24 = 11,520,000 bits
÷ 8 = 1,440,000 bytes
÷ 1,024 = 1,406.25 KB
÷ 1,024 = 1.37 MB (uncompressed)

Sound File Size

Digital audio is captured by taking regular samples. The file size depends on how many samples per second (sample rate), how many bits each sample uses (bit depth), and the duration.

Sound file size (bits) = Sample rate (Hz) × Bit depth (bits) × Duration (seconds) × Channels
(Channels: 1 = mono, 2 = stereo)

Worked Example — Sound

A 3-minute stereo sound file, sample rate 44,100 Hz, bit depth 16 bits. Calculate size in MB.

Duration = 3 × 60 = 180 seconds
Size = 44,100 × 16 × 180 × 2 = 254,016,000 bits
÷ 8 = 31,752,000 bytes
÷ 1,024 = 31,007.8 KB
÷ 1,024 = 30.3 MB (uncompressed)

Text File Size

Text file size (bits) = Number of characters × Bits per character
ASCII = 7 or 8 bits per character; Unicode = 16 or 32 bits per character

Why Units Matter

Storage manufacturers sometimes use SI units (1,000-based) to make their products seem larger: a "500 GB" hard drive may only show as ~465 GB in your OS (which uses 1,024-based units). This is a common source of confusion.

Common Mistakes

  • Using 1,000 instead of 1,024 for conversions in exam answers
  • Forgetting to multiply by the number of channels when calculating sound file size
  • Confusing bits (b) with bytes (B) — 1 byte = 8 bits
  • Forgetting colour depth is in BITS not bytes
  • Not converting duration to seconds before calculating sound file size
