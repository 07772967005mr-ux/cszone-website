1.3.1c — The Internet & DNS
Component 1: Computer Systems  ·  ~20 min

The Internet vs The World Wide Web

These two terms are NOT the same:

The Internet

A global network of interconnected networks (a WAN). The physical infrastructure — cables, routers, servers — that connects billions of devices worldwide. The internet existed before the World Wide Web.

The World Wide Web (WWW)

A service that runs on top of the internet. It is a collection of web pages and resources accessed via URLs using the HTTP/HTTPS protocol. Invented by Tim Berners-Lee in 1989.

The internet = the physical network infrastructure. The WWW = a service that runs on it (like email, FTP, and VoIP are also services that run on the internet).

IP Addresses

Every device on the internet has a unique IP (Internet Protocol) address. This is used to identify and locate it for routing data.

IPv4 vs IPv6

  • IPv4: 32-bit address written as 4 groups of decimal numbers (e.g. 192.168.1.1). About 4.3 billion possible addresses.
  • IPv6: 128-bit address written in hexadecimal (e.g. 2001:0db8:85a3:0000:0000:8a2e:0370:7334). Vastly more addresses — needed because IPv4 addresses are running out.

DNS — Domain Name System

Humans remember domain names (like www.google.com) but computers communicate using IP addresses. DNS is a system that translates domain names into IP addresses.

How DNS works (step by step):

1. User types www.google.com in browser
2. Device checks its local DNS cache first
3. If not found, device sends query to a DNS server
4. DNS server looks up the domain and returns the IP address
5. Browser uses the IP address to connect to Google's server
6. Web page loads
DNS = the internet's "phone book" — it converts human-readable domain names into machine-readable IP addresses.

Cloud Computing

Cloud computing means using remote servers (hosted on the internet) to store, manage, and process data, instead of local hardware.

Examples of cloud services:

  • Storage: Google Drive, iCloud, OneDrive, Dropbox
  • Software: Google Docs, Microsoft 365 (online)
  • Servers: Amazon Web Services (AWS), Microsoft Azure
  • Streaming: Netflix, Spotify (data processed on remote servers)

Advantages of cloud:

  • Access files from any device with internet access
  • No need to buy and maintain physical storage/servers
  • Files automatically backed up by provider
  • Scalable — pay for more storage/computing as needed
  • Always up-to-date software versions

Disadvantages of cloud:

  • Requires internet connection — cannot access offline
  • Data stored on third-party servers — privacy and security concerns
  • Recurring subscription costs
  • If provider goes offline, you lose access to your data

Common Mistakes

  • Saying the internet and WWW are the same thing — they are not
  • Saying DNS converts IP addresses to domain names — it does the opposite (domain name → IP address)
  • Forgetting that the cloud still uses physical servers — the term "cloud" is a metaphor
