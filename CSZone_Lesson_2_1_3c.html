2.1.3c — Bubble Sort
Component 2: Programming  ·  ~20 min

What is Bubble Sort?

Bubble sort repeatedly compares adjacent pairs of elements and swaps them if they are in the wrong order. Larger values gradually "bubble up" to the end of the list with each pass.

Key fact: After each pass, the largest unsorted value is in its correct final position. With n items, at most n−1 passes are needed.

How Bubble Sort Works

Step-by-step:

  1. Start at the first pair (index 0 and 1)
  2. Compare adjacent elements
  3. If left > right → swap them
  4. Move to the next adjacent pair
  5. Repeat to end of unsorted portion — this completes one pass
  6. Repeat passes until no swaps occur in a full pass (sorted)

OCR Pseudocode:

function bubbleSort(list) n = len(list) swapped = true while swapped == true swapped = false for i = 0 to n - 2 if list[i] > list[i+1] then temp = list[i] list[i] = list[i+1] list[i+1] = temp swapped = true endif next i endwhile return list endfunction

Example trace — sorting [5, 3, 1, 4, 2]:

Pass 1: Compare 5>3→swap [3,5,1,4,2]; 5>1→swap [3,1,5,4,2]; 5>4→swap [3,1,4,5,2]; 5>2→swap [3,1,4,2,5]
Pass 2: 3>1→swap [1,3,4,2,5]; 3<4→no swap; 4>2→swap [1,3,2,4,5]; 4<5→no swap
Pass 3: 1<3→no swap; 3>2→swap [1,2,3,4,5]; 3<4→no swap
Pass 4: No swaps → sorted: [1, 2, 3, 4, 5]

Optimisation: Early Termination

The swapped flag is an important optimisation. If an entire pass completes with no swaps, the list is already sorted — there is no need to continue. This allows bubble sort to exit early for nearly-sorted data.

Without optimisation: Always does n−1 passes = O(n²) always.
With optimisation: Best case O(n) — one pass on already-sorted list confirms no swaps needed.

Common Mistakes

  • Forgetting the swapped flag — without it, the algorithm always runs all n−1 passes even when sorted
  • Using 'n-1' instead of 'n-2' in the loop — list[i+1] must not go out of bounds when i is at the last index
  • Forgetting the temp variable for swapping — you MUST use a temp to avoid overwriting a value
  • Saying bubble sort is efficient — it is O(n²) worst case, one of the LEAST efficient sorting algorithms

Advantages and Disadvantages

AdvantagesDisadvantages
Simple to understand and implementInefficient — O(n²) worst case
Can detect if list is already sorted (with swapped flag)Not suitable for large datasets
In-place sorting — no extra memory neededMuch slower than merge sort for large data
