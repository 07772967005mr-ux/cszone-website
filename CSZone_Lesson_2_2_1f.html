2.2.1f — Subroutines
Component 2: Programming  ·  ~22 min

What is a Subroutine?

A subroutine is a named block of code that performs a specific task. It can be called (executed) from elsewhere in the program. There are two types: procedures and functions.

Key benefit: Subroutines avoid code repetition, make programs easier to read, test, and maintain. A subroutine written once can be called many times.

Procedures vs Functions

ProcedureFunction
Returns a value?NoYes — using return
OCR keywordprocedure / endprocedurefunction / endfunction + return
Use casePerforming an action (e.g. print menu)Calculating and returning a value

Procedure — no return value:

procedure greet(name) print("Hello, " + name) endprocedure greet("Alice") // call the procedure greet("Bob") // call again with different argument

Function — returns a value:

function square(n) return n * n endfunction result = square(5) // result = 25 print(square(3)) // prints 9

Parameters and Arguments

A parameter is a variable listed in a subroutine's definition — it receives the value passed in. An argument is the actual value passed when calling the subroutine.

// Parameters: a, b are PARAMETERS (in the definition) function add(a, b) return a + b endfunction // Arguments: 3 and 7 are ARGUMENTS (at the call) total = add(3, 7) // total = 10
Local variables: Variables declared inside a subroutine are LOCAL — they only exist within that subroutine and cannot be accessed from outside it.

Common Mistakes

  • Using 'function' when no value is returned — use 'procedure' instead
  • Forgetting 'return' in a function — without it, no value is sent back to the caller
  • Confusing parameter (variable in definition) with argument (value at call site)
  • Accessing a local variable outside the subroutine where it was defined
  • Not calling the subroutine — defining it does nothing; it must be called to execute
