2.2.1d — File Handling
Component 2: Programming  ·  ~18 min

Why File Handling?

Variables only store data while a program runs — data is lost when the program ends. File handling allows programs to persistently store data to secondary storage so it can be retrieved in future sessions.

Persistent storage: Data saved to a file survives after the program ends. Variables (RAM) do not.

OCR Pseudocode: Four File Operations

1. OPEN — open a file for reading or writing:

open("scores.txt", "r") // open for reading open("scores.txt", "w") // open for writing (overwrites existing) open("scores.txt", "a") // open for appending (adds to end)

2. READ — read data from a file:

open("data.txt", "r") line = readline("data.txt") // read one line close("data.txt")

3. WRITE — write data to a file:

open("output.txt", "w") writeline("output.txt", "Hello World") close("output.txt")

4. CLOSE — close the file when finished:

close("scores.txt") // always close after use
Always close: Failing to close a file can cause data loss or corruption. It also releases the file for other programs to use.

Reading All Lines with EOF

Reading until End Of File (EOF):

open("names.txt", "r") while NOT endOfFile("names.txt") name = readline("names.txt") print(name) endwhile close("names.txt")

Writing multiple lines:

scores = [85, 72, 91, 68] open("results.txt", "w") for i = 0 to len(scores) - 1 writeline("results.txt", str(scores[i])) next i close("results.txt")

Common Mistakes

  • Forgetting to close the file — always call close() after use
  • Opening in "w" mode when you want to append — "w" OVERWRITES; use "a" to add to existing file
  • Not checking EOF before reading — reading past end causes an error
  • Forgetting that readline() returns a STRING — convert with int() if you need a number
