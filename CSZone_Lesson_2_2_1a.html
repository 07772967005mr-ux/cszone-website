2.2.1a — Variables & Data Types
Component 2: Programming  ·  ~20 min  ·  FREE

Variables and Constants

A variable is a named location in memory used to store data that can change during a program's execution. A constant stores a value that does not change during execution.

OCR Pseudocode examples:

name = "Alice" // variable — can change age = 16 // variable — integer PI = 3.14159 // constant — does not change score = score + 10 // updating a variable
Assignment operator: In OCR pseudocode, = is used for assignment (storing a value). == is used for comparison (testing equality).

Data Types

Every variable has a data type that determines what kind of data it stores and what operations are valid on it.

Data TypeDescriptionExample
IntegerWhole number (positive or negative)42, -5, 0
Real / FloatNumber with decimal point3.14, -0.5, 2.0
Booleantrue or false onlytrue, false
StringSequence of characters (text)"Hello", "A1"
CharacterA single character'A', '5', '!'
Note: OCR refers to decimals as 'real' numbers. Python uses 'float'. Both terms may appear in exam questions.

Arithmetic Operators

OCR Pseudocode arithmetic operators:

MOD uses: Testing if even (n MOD 2 == 0), checking divisibility, wrapping around indices.

Input and Output

name = input("Enter your name: ") // reads from keyboard print("Hello " + name) // outputs to screen age = int(input("Age: ")) // convert string input to integer

Common Mistakes

  • Confusing = (assignment) with == (comparison): x = 5 stores 5; x == 5 tests if x equals 5
  • Using integer when decimal needed — dividing 7/2 gives 3.5 (real), not 3 (integer DIV)
  • Forgetting to convert input() — it always returns a string; use int() or float() for numbers
  • Confusing character (one symbol, single quotes) with string (any length, double quotes)
