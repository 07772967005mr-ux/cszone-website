1.3.1b — Network Types
Component 1: Computer Systems  ·  ~20 min

Client-Server Networks

In a client-server network, one or more powerful computers (servers) provide services and resources to other computers (clients) on the network.

How it works:

  • A server stores files, manages access rights, handles printing, hosts websites, etc.
  • Clients request services from the server (e.g. a student's PC asking the school server for a file)
  • The server responds to requests and controls access (authentication, permissions)

Advantages of client-server:

  • Centralised data — easy to back up and manage from one place
  • Centralised security — access control managed on the server
  • Files accessible from any client on the network
  • Easier to update software — update once on the server

Disadvantages of client-server:

  • If the server fails, the whole network is affected
  • Expensive to set up and maintain (specialist hardware, IT staff)
  • Can become slow if many clients request services simultaneously

Peer-to-Peer (P2P) Networks

In a peer-to-peer network, all computers have equal status. Each device can act as both a client and a server — sharing its own resources and accessing others' resources.

Advantages of P2P:

  • Easy and cheap to set up — no specialist server required
  • If one computer fails, others are unaffected
  • No dependence on a central server
  • Good for small networks (e.g. home)

Disadvantages of P2P:

  • No centralised management — harder to control access and security
  • Each computer manages its own backups
  • Performance degrades as more devices connect
  • Files may be duplicated across devices

Virtual Networks

A virtual network (or VLAN — Virtual Local Area Network) is a logical grouping of devices within a physical network, creating separate networks without separate hardware.

Key points:

  • Devices in the same VLAN can communicate as if on the same LAN, even if physically separate
  • Improves security — different departments can be on separate VLANs (e.g. finance and HR)
  • Reduces broadcast traffic (data is only sent to devices in the same VLAN)
  • Flexible — easy to change without recabling

Common Mistakes

  • Saying a P2P network has no files shared — files are shared, just not from a central server
  • Confusing the server in client-server with a physical server machine (servers can run on powerful PCs too)
  • Saying client-server is always better — P2P is often better for small/home networks
  • Confusing VLAN with a separate physical network — a VLAN is a logical separation, not physical
