1.3.2b — Network Protocols
Component 1: Computer Systems  ·  ~22 min

What is a Protocol?

A protocol is a set of rules that govern how data is transmitted across a network. Protocols ensure devices from different manufacturers can communicate — this is called interoperability.

The TCP/IP Model

TCP/IP is a layered model that describes how data is sent over the internet. OCR J277 requires understanding of the four key layers:

LayerNamePurposeProtocols
4 (top)ApplicationProvides network services to applications (email, web, file transfer)HTTP, HTTPS, FTP, SMTP, IMAP, DNS
3TransportBreaks data into segments (packets), ensures reliable deliveryTCP, UDP
2InternetLogical addressing (IP addresses), routing between networksIP
1 (bottom)LinkPhysical transmission of data on the local network (MAC addresses)Ethernet, Wi-Fi
Memory trick: Application → Transport → Internet → Link (All The Important Layers)

Key Protocols

HTTP / HTTPS

  • HyperText Transfer Protocol — transfers web pages
  • HTTPS = HTTP + TLS/SSL encryption (secure)
  • Port 80 (HTTP), Port 443 (HTTPS)

FTP (File Transfer Protocol)

  • Transfers files between computers over a network
  • Used by developers to upload files to web servers
  • Port 20 (data), Port 21 (control)

SMTP / IMAP / POP3 (Email)

  • SMTP — Simple Mail Transfer Protocol: SENDS email (port 25/587)
  • IMAP — Internet Message Access Protocol: retrieves email from server, emails STAY on server
  • POP3 — Post Office Protocol 3: downloads email to device, REMOVES from server

TCP vs UDP

  • TCP (Transmission Control Protocol): reliable, ordered delivery. Data split into packets, sent, receiver acknowledges (ACK). Lost packets resent. Used for: web browsing, email, file downloads.
  • UDP (User Datagram Protocol): fast, no guarantee of delivery. No acknowledgement. Used for: video streaming, online gaming, VoIP — speed matters more than perfect accuracy.

IP (Internet Protocol)

  • Provides logical addressing (IP addresses) for devices
  • Routes packets across networks (via routers)
  • IPv4 (32-bit, ~4.3 billion addresses) vs IPv6 (128-bit, vastly more)

Packet Switching

Data is broken into small chunks called packets. Each packet contains:

  • Header: source IP, destination IP, packet number, total packets
  • Payload: the actual data chunk
  • Trailer: error checking information (checksum)

Packets travel independently across the network — possibly taking different routes. At the destination, they are reassembled in order.

Advantages of packet switching:

  • If a router fails, packets take alternative routes
  • Network resources are shared efficiently
  • Large files are broken into manageable chunks

Common Mistakes

  • Confusing SMTP (sends) with IMAP/POP3 (retrieves) — SMTP = sending, IMAP/POP3 = receiving
  • Saying TCP is always better — UDP is preferred for real-time applications where a tiny bit of lost data is acceptable
  • Forgetting packets include headers and trailers — not just raw data
  • Mixing up IMAP and POP3 — IMAP keeps emails on server, POP3 removes them
