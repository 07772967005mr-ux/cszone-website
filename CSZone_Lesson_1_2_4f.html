1.2.4f — Representing Images
Component 1: Computer Systems  ·  ~20 min

Pixels and Bitmaps

Digital images are made up of tiny squares called pixels (picture elements). A bitmap image stores the colour of every single pixel as a binary number. The more pixels, the higher the resolution and detail.

Bitmap = a grid of pixels, each stored as a binary number representing its colour.

Colour Depth

Colour depth (also called bit depth) is the number of bits used to represent the colour of each pixel. More bits = more possible colours.

Colour depthColours possibleExample
1 bit2¹ = 2Black and white only
4 bits2⁴ = 16Basic colours
8 bits2⁸ = 256256-colour palette
24 bits2²⁴ = 16,777,216True colour (8 bits each for R, G, B)

24-bit True Colour

24-bit images use 8 bits each for Red, Green, and Blue channels. 8 bits per channel = 256 shades each = 256³ = 16,777,216 colours. This is what most photos use.

Image Resolution

Resolution is the number of pixels in an image, usually given as width × height (e.g. 1920×1080). Higher resolution = more pixels = more detail = larger file size.

Resolution vs Colour Depth

Both affect image quality and file size. Increasing either one increases the file size.

Calculating Image File Size

Formula: File size (bits) = Width × Height × Colour depth
Divide by 8 to convert to bytes.

Example 1: 100×200 image, 8-bit colour

File size = 100 × 200 × 8 bits
= 160,000 bits
= 160,000 ÷ 8 = 20,000 bytes
= 20,000 ÷ 1024 ≈ 19.5 KB

Example 2: 1920×1080 image, 24-bit colour

File size = 1920 × 1080 × 24 bits
= 49,766,400 bits
= 49,766,400 ÷ 8 = 6,220,800 bytes
≈ 6.22 MB (uncompressed)

Common Mistakes

  • Forgetting to divide by 8 to convert bits to bytes
  • Using KB = 1000 instead of 1024 (in computing, 1 KB = 1024 bytes)
  • Confusing resolution (pixel count) with colour depth (bits per pixel)
  • Forgetting to multiply width × height first to get total pixel count
  • Saying higher colour depth = more pixels (it does NOT — it means more colours per pixel)

Metadata

Image files also contain metadata — information about the image stored with the file but not part of the visible image. This includes width, height, colour depth, file format, date taken, camera settings, GPS location, etc.

Metadata is not part of the image content — it describes the image. Total file size = image data + metadata.

Effect of Changing Resolution and Colour Depth

ChangeEffect on qualityEffect on file size
Increase resolution (more pixels)More detail, sharperIncreases proportionally
Decrease resolutionLess detail, pixelatedDecreases
