1.1.3 Embedded Systems

Subtopic 3 of 3 · Component 1: Computer Systems

What is an Embedded System?

An embedded system is a computer system (with a processor, memory, and input/output) that is built into a larger device to perform a specific, dedicated function. Unlike a general-purpose computer (such as a laptop or desktop PC), an embedded system is designed to do one particular job and does it very well.

Embedded systems are found in almost every electronic device around us — from the microwave in your kitchen to the engine management system in a car. The processor is typically hidden inside the device and the user often doesn't even know it's there.

💡 Key Definition An embedded system is a computer system embedded within a larger device, designed to perform one specific/dedicated function, often in real time.

Characteristics of Embedded Systems

Embedded systems share several key characteristics that distinguish them from general-purpose computers:

1. Single/Dedicated Function

Each embedded system is built to perform one specific task. A washing machine controller doesn't browse the internet or play games — it only controls wash cycles. This focus makes embedded systems more reliable and efficient for their specific purpose.

2. Real-Time Operation

Many embedded systems must respond to inputs immediately. A car's antilock braking system (ABS) must react to wheel sensors in milliseconds — a delay could be fatal. This is called real-time processing. The system must guarantee a response within a fixed time period.

3. Limited Resources

Embedded systems typically have:

  • Limited processing power — they only need enough to do their specific task
  • Limited memory (RAM) — often kilobytes, not gigabytes
  • Small storage — the program is often stored in ROM (read-only memory) and doesn't change

This is deliberate — using only what's needed keeps costs low and power consumption minimal.

4. Low Power Consumption

Many embedded systems run on batteries (e.g. pacemakers, thermostats) or must be energy-efficient. Because they only do one job, they can be optimised to use very little power.

5. The Program is Fixed (Not Easily Reprogrammed)

The software in an embedded system is usually written once and stored permanently in ROM (or Flash memory). Unlike a PC where you install and uninstall software freely, an embedded system's program rarely changes — it is embedded with the hardware.

⚠️ Exam tip Questions often ask you to "give an example of an embedded system and explain its function." Give a specific device (e.g. "ABS in a car") and then explain what it actually does ("it monitors wheel speed and automatically applies individual brakes to prevent skidding on icy roads").
⚠️ Common Mistakes
  • ❌ "A smartphone is an embedded system" — smartphones are general-purpose. However, a specific chip inside a phone (e.g. its GPS receiver or camera sensor processor) IS an embedded system
  • ❌ "Embedded systems have no memory" — they have limited memory, not no memory
  • ❌ Confusing "real-time" with "fast" — real-time means guaranteed response time, not just quick
  • ❌ Not giving specific examples — always name the device AND its function
