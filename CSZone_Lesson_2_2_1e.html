2.2.1e — String Handling
Component 2: Programming  ·  ~20 min

String Operations in OCR Pseudocode

Strings are sequences of characters. OCR J277 requires knowledge of several string operations — these appear frequently in programming exam questions.

Strings are zero-indexed in OCR pseudocode — the first character is at position 0.

Key string operations — summary table:

Lengthlen(s)len("Hello")5

substring() Detail

The substring(s, start, length) function extracts part of a string. It takes three arguments: the string, the starting index (0-indexed), and the number of characters to extract.

word = "Computer" // 01234567 substring(word, 0, 4) // "Comp" — start at 0, take 4 chars substring(word, 3, 3) // "put" — start at 3, take 3 chars substring(word, 4, 4) // "uter" — start at 4, take 4 chars // Accessing individual characters (like an array): word[0] // "C" word[7] // "r" len(word) // 8

Common Mistakes

  • Treating strings as 1-indexed — they are zero-indexed: first character = index 0
  • Confusing substring(s, start, LENGTH) vs Python's s[start:end] — OCR uses length, not end index
  • Concatenating a number directly: "Score: " + score — must convert with str(score) first
  • Forgetting upper()/lower() for case-insensitive comparisons in validation
