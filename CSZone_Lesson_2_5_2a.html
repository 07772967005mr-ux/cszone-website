2.5.2a — Translators
Component 2: Languages & IDEs  ·  ~22 min

Why Do We Need Translators?

The CPU can only execute machine code (binary). High-level and assembly programs must be converted to machine code before they can run. Translators perform this conversion.

Three types of translator: Compiler, Interpreter, Assembler — each converts code differently.

Assembler

An assembler translates assembly language code (mnemonics) into machine code. Each assembly instruction is converted to its binary equivalent — one-to-one. Covered in 2.5.1.

Compiler

A compiler translates the entire high-level source code program into machine code before execution. The output is a standalone executable file (object code) that can be run without the compiler.

Advantages of compilers:

  • The compiled executable runs FASTER — no translation needed at runtime
  • Source code is protected — users receive only the compiled binary, not readable source
  • Compiled code can be distributed and run without the compiler installed
  • All errors are reported at once (after compilation) — can fix in one pass

Disadvantages of compilers:

  • Compilation takes time — the full program must be translated before testing
  • Compiled code is platform-specific — must recompile for different operating systems
  • If an error exists, you fix it and must recompile the entire program

Examples: C, C++, Rust use compilers. Java compiles to bytecode.

Interpreter

An interpreter translates and executes source code line by line, at runtime. It does NOT produce a standalone executable — the interpreter must be present every time the program runs.

Advantages of interpreters:

  • Easier to debug — errors are reported at the exact line where they occur
  • Platform-independent — the same source code runs on any platform with the interpreter
  • Faster to test during development — no compilation step needed
  • Useful for scripting and rapid prototyping

Disadvantages of interpreters:

  • Slower execution — translation happens at runtime, every time the program runs
  • The interpreter must be installed on every machine that runs the program
  • Source code is exposed — easier for users to read or copy
  • Stops at the first error found — subsequent errors not shown until earlier ones are fixed

Examples: Python, JavaScript, Ruby are typically interpreted.

Common Mistakes

  • Saying interpreters produce an executable file — they do NOT. Compilers produce executables; interpreters translate line-by-line at runtime
  • Saying compilers are slower than interpreters — compiled programs RUN faster; interpreters are slower at runtime
  • Saying interpreters show all errors at once — they STOP at the first error; compilers report all errors after full compilation
  • Confusing assembler with compiler — assembler handles ASSEMBLY; compiler handles HIGH-LEVEL languages
