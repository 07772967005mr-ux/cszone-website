2.1.1 — Computational Thinking
Component 2: Programming  ·  ~18 min  ·  Free lesson

What is Computational Thinking?

Computational thinking is a problem-solving approach that involves breaking problems down and expressing solutions in a way that a computer can carry out. It comprises four key concepts that are heavily tested in OCR J277.

The four pillars: Decomposition · Abstraction · Algorithmic thinking · Pattern recognition

The Four Pillars

1. Decomposition

Breaking a complex problem down into smaller, more manageable sub-problems. Each sub-problem can then be solved independently and combined to solve the overall problem.

Example: Building a social media app → decompose into: user login, news feed, messaging, notifications, profile management.

2. Abstraction

Focusing only on the important details of a problem and ignoring irrelevant information. Hiding unnecessary complexity to simplify the problem.

Example: A map is an abstraction of the real world — it shows roads and buildings but ignores tree species, soil composition, etc. A programmer uses a function without needing to know exactly how it works internally.

3. Algorithmic Thinking

Developing a step-by-step solution (algorithm) that can be followed to solve the problem. The algorithm must be precise — a computer executes instructions exactly as written.

Example: Writing the exact steps to sort a list of numbers — checking each pair, swapping if out of order, repeating until sorted (bubble sort).

4. Pattern Recognition

Identifying similarities, trends, or patterns within and between problems. Recognising that the same type of solution can be applied to similar problems avoids reinventing the wheel.

Example: Recognising that all sorting algorithms follow a similar pattern of comparing and rearranging elements. Or recognising that a login system on a website uses the same validation pattern as other apps.

Why is Computational Thinking Important?

These techniques allow programmers to approach ANY complex problem systematically. Decomposition ensures nothing is missed; abstraction prevents information overload; algorithmic thinking produces precise solutions; pattern recognition allows reuse of existing solutions.

Common Mistakes

  • Confusing abstraction with decomposition — abstraction removes irrelevant detail; decomposition breaks into sub-problems
  • Defining abstraction as "hiding data" — it's specifically about removing irrelevant details to focus on what matters
  • Forgetting to include an example when explaining each concept
  • Saying "pattern recognition is finding repeated code" — it's identifying similarities in problems, not just code
