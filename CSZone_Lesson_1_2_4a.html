1.2.4a — Binary & Denary Conversion
Component 1: Computer Systems  ·  ~25 min

Number Systems

Computers use the binary (base 2) number system because electronic components have two states: on (1) or off (0). Humans use denary (base 10, also called decimal) which has 10 digits: 0–9.

Notation: To avoid confusion, binary numbers are often written with subscript 2 (e.g. 1010₂) and denary with subscript 10 (e.g. 10₁₀).

The most significant bit (MSB) is on the left (128), the least significant bit (LSB) is on the right (1).

An 8-bit number can represent values from 0 to 255 (2⁸ − 1 = 255).

Converting Binary to Denary

Write out the place values, then add up the values where the binary digit is 1.

Worked Example: Convert 10110101₂ to denary

128 | 64 | 32 | 16 | 8 | 4 | 2 | 1
 1  |  0  |  1  |  1  | 0 | 1 | 0 | 1

= 128 + 32 + 16 + 4 + 1
= 181

Worked Example: Convert 01001100₂ to denary

128 | 64 | 32 | 16 | 8 | 4 | 2 | 1
 0  |  1  |  0  |  0  | 1 | 1 | 0 | 0

= 64 + 8 + 4
= 76

Converting Denary to Binary

Method: Work from the largest place value (128) downward. If the remaining value is ≥ the place value, put a 1 and subtract; otherwise put a 0.

Worked Example: Convert 197 to 8-bit binary

197 ≥ 128? YES → put 1, remainder = 197 − 128 = 69
69 ≥ 64?  YES → put 1, remainder = 69 − 64 = 5
5 ≥ 32?   NO  → put 0, remainder = 5
5 ≥ 16?   NO  → put 0, remainder = 5
5 ≥ 8?    NO  → put 0, remainder = 5
5 ≥ 4?    YES → put 1, remainder = 5 − 4 = 1
1 ≥ 2?    NO  → put 0, remainder = 1
1 ≥ 1?    YES → put 1, remainder = 0

Result: 11000101₂

Worked Example: Convert 45 to 8-bit binary

45 ≥ 128? NO → 0  | 45 ≥ 64? NO → 0  | 45 ≥ 32? YES → 1, rem=13
13 ≥ 16? NO → 0  | 13 ≥ 8? YES → 1, rem=5  | 5 ≥ 4? YES → 1, rem=1
1 ≥ 2? NO → 0  | 1 ≥ 1? YES → 1
Result: 00101101₂
Check: Always verify by converting your binary answer back to denary!

Key Facts to Remember

  • 8-bit binary: 0 to 255 (2⁸ values)
  • With n bits, you can represent 2ⁿ different values (0 to 2ⁿ − 1)
  • The MSB (leftmost bit) has the highest value (128 for 8-bit)
  • The LSB (rightmost bit) always has value 1
  • All 1s in 8-bit = 11111111₂ = 255₁₀
  • All 0s = 00000000₂ = 0₁₀

Common Mistakes

  • Starting place values from the wrong end (always start with 128 on the left)
  • Forgetting leading zeros in 8-bit representation (e.g. 45 = 00101101, not 101101)
  • Adding up all place values instead of only those with a 1
  • Confusing binary (base 2) with denary (base 10)
