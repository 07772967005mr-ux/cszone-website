2.3.1 — Defensive Design
Component 2: Programming  ·  Free lesson  ·  ~20 min

What is Defensive Design?

Defensive design means writing programs that anticipate and handle errors before they cause crashes or incorrect results. A well-designed program should deal gracefully with unexpected, invalid, or malicious inputs.

OCR says: Defensive design considers: input validation, input sanitisation, planning for contingencies, anticipating misuse, and using authentication.

Input Validation

Validation checks that input data is sensible, reasonable, and within acceptable bounds — BEFORE it is processed by the program. Validation does NOT check whether data is accurate (correct), only that it meets the expected rules.

Validation typeWhat it checksExample
Range checkValue is within acceptable min/maxAge must be 0–120
Type checkData is the correct type (integer, string…)Age must be an integer
Length checkString length is within acceptable boundsPassword must be 8–20 chars
Presence checkField is not empty (required)Username field must not be blank
Format checkData matches an expected patternPostcode matches AB1 2CD pattern
Look-up checkValue exists in a known listCountry selected from dropdown list

Validation loop in pseudocode:

age = input("Enter age: ") while age < 0 OR age > 120 print("Invalid age. Enter 0-120.") age = input("Enter age: ") endwhile

Input Sanitisation

Sanitisation cleans or transforms input so that it cannot cause harm — by removing or replacing unwanted characters. It is different from validation: validation REJECTS bad input; sanitisation CLEANS it.

Examples of sanitisation:

  • Stripping HTML/script tags from user input to prevent cross-site scripting (XSS)
  • Removing special characters from a database query to prevent SQL injection
  • Converting a string to lowercase before comparing (so "Admin" == "admin")
  • Trimming whitespace from start/end of a string

Authentication

Authentication verifies the identity of a user before granting access to a system or resource. It ensures only authorised users can use the software.

Common authentication methods:

  • Username + password — most common. Password stored as a hash, not plain text.
  • Two-factor authentication (2FA) — password plus a second factor (e.g. OTP via SMS or app)
  • Biometric — fingerprint, face recognition

Authentication loop in pseudocode:

username = input("Username: ") password = input("Password: ") if username == "admin" AND password == "secret123" print("Access granted") else print("Access denied") endif

Anticipating Misuse

Good programmers think like an attacker — they consider all the ways a user might misuse the program and add checks to prevent problems.

Examples of misuse to anticipate:

  • Entering letters where numbers are expected
  • Entering negative numbers for fields that must be positive
  • Leaving required fields blank
  • Attempting to access unauthorised areas of the system
  • Buffer overflow by entering extremely long strings
  • Entering SQL or script code as input

Common Mistakes

  • Confusing validation (checking data) with sanitisation (cleaning data)
  • Saying validation checks if data is "correct/accurate" — it only checks if it's reasonable/within rules
  • Forgetting to re-ask for input inside a validation loop — just showing an error message is insufficient
  • Confusing authentication (who you are) with authorisation (what you're allowed to do)
  • Thinking defensive design only means validation — it also includes sanitisation, authentication, and error handling
