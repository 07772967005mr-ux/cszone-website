2.2.1b — Selection & Iteration
Component 2: Programming  ·  ~22 min

Selection — IF Statements

Selection allows a program to make decisions — executing different code depending on whether a condition is true or false.

IF / ELSEIF / ELSE:

if score >= 70 then print("Grade A") elseif score >= 50 then print("Grade B") elseif score >= 30 then print("Grade C") else print("Fail") endif

Comparison operators used in conditions:

== equal to  ·  != not equal to  ·  < less than  ·  > greater than  ·  <= less than or equal  ·  >= greater than or equal

Boolean operators:

if age >= 16 AND hasID == true then print("Admitted") endif if x < 0 OR x > 100 then print("Out of range") endif if NOT active then print("Inactive") endif

Iteration — Loops

Iteration allows code to be repeated. OCR J277 requires three types of loop.

FOR loop — count-controlled (known number of iterations):

for i = 1 to 5 print(i) next i // prints 1, 2, 3, 4, 5

WHILE loop — condition-controlled (pre-condition: checked BEFORE each iteration):

x = 1 while x <= 5 print(x) x = x + 1 endwhile // may execute 0 times if condition is initially false

DO UNTIL loop — post-condition (checked AFTER each iteration — always runs at least once):

do password = input("Enter password: ") until password == "secret" // always runs at least once
Key difference: WHILE checks condition BEFORE (may never run). DO UNTIL checks AFTER (always runs at least once). FOR is used when number of iterations is known.

Common Mistakes

  • Forgetting endif / endwhile / next i — missing closing keywords cause syntax errors
  • Using = instead of == in a condition: if x = 5 should be if x == 5
  • Infinite loop — if a WHILE condition never becomes false, the loop runs forever
  • Confusing WHILE (pre-condition) and DO UNTIL (post-condition) — only DO UNTIL is guaranteed to run once
  • Off-by-one errors — for i = 1 to 5 gives 5 iterations; for i = 0 to 4 also gives 5 iterations
