2.1.3e — Insertion Sort
Component 2: Programming  ·  ~18 min

What is Insertion Sort?

Insertion sort builds a sorted list one element at a time. It takes each element and inserts it into the correct position within the already-sorted portion of the list — like sorting a hand of playing cards.

Key analogy: Think of sorting a hand of cards. You pick up each new card and slide it left until it's in the right place among the cards already sorted in your hand.

How Insertion Sort Works

Step-by-step:

  1. Start from index 1 (the first element, index 0, is trivially sorted)
  2. Take the current element as the 'key'
  3. Compare key with elements to its left (in the sorted portion)
  4. Shift larger elements one position to the right
  5. Insert the key in the correct position
  6. Move to the next element and repeat

OCR Pseudocode:

function insertionSort(list) for i = 1 to len(list) - 1 key = list[i] j = i - 1 while j >= 0 AND list[j] > key list[j+1] = list[j] j = j - 1 endwhile list[j+1] = key next i return list endfunction

Full trace — [5, 2, 4, 1, 3]:

i=1, key=2: 5>2 → shift 5 right → insert 2 → [2,5,4,1,3]
i=2, key=4: 5>4 → shift 5 right; 2<4 → stop → insert 4 → [2,4,5,1,3]
i=3, key=1: 5>1→shift; 4>1→shift; 2>1→shift; j=-1→stop → insert 1 → [1,2,4,5,3]
i=4, key=3: 5>3→shift; 4>3→shift; 2<3→stop → insert 3 → [1,2,3,4,5] ✓

Advantages and Disadvantages

AdvantagesDisadvantages
Efficient for small or nearly-sorted listsO(n²) worst case — slow for large unsorted lists
Simple to implement and understandLess efficient than merge sort for large data
In-place — O(1) extra memoryQuadratic number of comparisons on reverse-sorted input
Best case O(n) on already-sorted listsNot practical for large randomly-ordered datasets

Common Mistakes

  • Starting the outer loop at index 0 — the first element is already 'sorted'; start at index 1
  • Forgetting to shift elements right rather than swap — insertion sort shifts, not swaps
  • Saying insertion sort is O(n log n) — it is O(n²) worst case
  • Confusing insertion sort with selection sort — selection sort SELECTS the minimum and places it; insertion sort takes the next element and INSERTS it
