1.4.2 — Vulnerabilities & Security Measures
Component 1: Computer Systems  ·  ~22 min

Causes of Vulnerabilities

Systems can have weaknesses that attackers exploit. Understanding causes helps with prevention.

Software vulnerabilities:

  • Unpatched software — outdated software with known bugs that haven't been fixed
  • Weak passwords — easy to guess or brute-force; no lockout policies
  • Poor input validation — allows SQL injection, buffer overflow attacks
  • Misconfigured systems — default passwords, unnecessary open ports

People vulnerabilities:

  • Falling for social engineering (phishing, blagging)
  • Using the same password for multiple accounts
  • Not logging out of accounts on shared devices
  • Connecting to unsecured public Wi-Fi

Security Measures

Penetration testing:

Authorised ethical hackers attempt to break into a system to find vulnerabilities before real attackers do. Identifies weaknesses so they can be patched.

Network forensics:

Monitoring and analysing network traffic to detect attacks, gather evidence, and understand how a breach occurred.

Firewalls:

  • Hardware or software that monitors and filters network traffic
  • Blocks unauthorised access based on rules
  • Can block specific IP addresses or ports

Encryption:

  • Scrambles data so it can only be read with the correct key
  • Protects data in transit (HTTPS/TLS) and at rest (disk encryption)
  • Even if intercepted, encrypted data is unreadable

Passwords & Authentication:

  • Strong passwords: long, mix of uppercase/lowercase/numbers/symbols
  • Two-Factor Authentication (2FA/MFA): second verification step (SMS code, authenticator app)
  • Biometrics: fingerprint, face recognition
  • Account lockout after failed attempts

Physical security:

  • Locked server rooms — restricts physical access to hardware
  • CCTV monitoring
  • Visitor logs and access cards

Anti-malware software:

  • Scans files and processes for known malware signatures
  • Must be kept up to date — new malware is released constantly
  • Quarantines or deletes detected threats

Regular updates/patches:

Software vendors release patches to fix known security vulnerabilities. Regular updating reduces the window of exposure to known exploits.

User access levels / principle of least privilege:

Users are only given the minimum access rights needed to do their job. Limits damage if an account is compromised.

Common Mistakes

  • Confusing a firewall with anti-malware — firewalls filter traffic; anti-malware scans files
  • Saying "use a strong password" without explaining WHAT makes it strong
  • Forgetting 2FA — very important and commonly tested
  • Not distinguishing between encryption in transit vs at rest
