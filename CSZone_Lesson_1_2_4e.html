1.2.4e — Character Encoding
Component 1: Computer Systems  ·  ~20 min

Why Does Character Encoding Exist?

Computers can only store numbers. To store text, each character must be assigned a unique number. A character encoding standard defines the mapping between characters and their numeric codes.

Key idea: Every letter, digit, symbol, and control character has a unique binary number assigned to it by a standard.

ASCII

ASCII (American Standard Code for Information Interchange) was developed in the 1960s. It uses 7 bits per character, giving 2⁷ = 128 possible characters.

ASCII includes:

  • Uppercase A–Z (65–90)
  • Lowercase a–z (97–122)
  • Digits 0–9 (48–57)
  • Punctuation and symbols (e.g. space = 32, ! = 33)
  • Control characters (e.g. newline = 10, tab = 9)

Pattern: A=65, B=66 ... a=97, b=98. Lowercase letters are exactly 32 more than uppercase (difference = 0b0100000). Digits start at 48.

ASCII Limitation

128 characters is not enough for the world's languages. ASCII has no characters for é, ñ, Chinese, Arabic, emoji, etc. This led to the development of Unicode.

Extended ASCII

Extended ASCII uses 8 bits (1 byte) per character = 256 characters. The extra 128 slots (128–255) were used for accented characters and symbols. However, different systems used different Extended ASCII variants, causing compatibility problems.

Unicode

Unicode was created to encode characters from all the world's writing systems in a single standard. It supports over 1 million possible code points.

Key Unicode facts:

  • The first 128 code points are identical to ASCII (backward compatible)
  • Supports Latin, Cyrillic, Arabic, Chinese, emoji, and more
  • Most commonly implemented as UTF-8 (variable length: 1–4 bytes per character)
  • UTF-8 uses 1 byte for ASCII characters, 2–4 bytes for other characters
  • UTF-16 uses 2 or 4 bytes per character
  • UTF-32 uses 4 bytes per character (fixed)
Unicode vs ASCII: Unicode uses more bits per character than ASCII (more storage), but can represent characters from all languages. UTF-8 is efficient because common English characters still only need 1 byte.

Common Mistakes

  • Confusing the character '0' (ASCII 48) with the number 0 (binary 00000000)
  • Saying Unicode uses 16 bits — UTF-8 uses 8 to 32 bits depending on the character
  • Forgetting ASCII has 128 characters (7 bits = 2⁷ = 128), not 256
  • Forgetting lowercase letters have ASCII codes 32 higher than uppercase
  • Saying Unicode replaced ASCII — Unicode's first 128 code points ARE identical to ASCII
