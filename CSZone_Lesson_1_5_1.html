1.5.1 — Operating Systems
Component 1: Computer Systems  ·  ~20 min  ·  Free lesson

What is an Operating System?

An Operating System (OS) is system software that manages hardware and software resources and provides an interface between the user and the computer. Without an OS, application software cannot run.

Examples: Windows 11, macOS Ventura, Linux Ubuntu, Android, iOS

Functions of an Operating System

1. Memory Management

The OS decides which programs get how much RAM. It allocates and deallocates memory as programs start and finish. Prevents one program accessing another's memory space.

2. Processor/CPU Scheduling

The OS manages which processes run on the CPU and when. Uses scheduling algorithms (e.g., round robin, priority scheduling) to give each process CPU time — creating the illusion of multitasking.

3. Peripheral Management (via device drivers)

The OS communicates with peripherals (printer, keyboard, monitor) using device drivers — software that translates OS instructions into device-specific commands.

4. File Management

The OS organises files in a hierarchical folder structure. It handles reading, writing, renaming, and deleting files. Controls permissions — who can access what files.

5. User Interface

  • GUI (Graphical User Interface): windows, icons, menus, pointer — visual and intuitive (Windows, macOS)
  • CLI (Command Line Interface): text-based commands — faster for experts, uses less memory (Linux terminal)

6. User Management

The OS handles multiple user accounts with different access rights and permissions. Ensures users can only access files and settings they are authorised for.

7. Security Management

The OS enforces login authentication, manages user permissions, and provides basic security features.

Common Mistakes

  • Saying the OS "runs programs" — it manages resources and provides an environment; the CPU runs programs
  • Forgetting device drivers — a key function of peripheral management
  • Confusing GUI and CLI — in the exam, CLI uses less memory/is faster for experts; GUI is more user-friendly
  • Leaving out CPU scheduling — a very commonly tested function
