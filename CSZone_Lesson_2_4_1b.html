2.4.1b — Truth Tables
Component 2: Boolean Logic  ·  ~22 min

What is a Truth Table?

A truth table shows every possible combination of inputs to a logic circuit and the corresponding output. It is used to analyse and design Boolean logic circuits.

Key rule: For n inputs, there are 2ⁿ rows (combinations). 1 input = 2 rows. 2 inputs = 4 rows. 3 inputs = 8 rows.

Building Truth Tables for Combined Circuits

When a circuit has multiple gates, add intermediate columns for each gate's output — working left to right through the circuit.

Example: NOT(A AND B)

Circuit: A and B → AND gate → NOT gate → Output

0101
1001
1110

Example: (A OR B) AND (NOT A)

Circuit: A → NOT gate → NOT_A; then A and B → OR gate → OR_out; then NOT_A and OR_out → AND gate → Final output

Method: Completing a Truth Table

  1. Count the inputs (n). Add 2ⁿ rows.
  2. Fill in input columns systematically: rightmost column alternates 0,1,0,1... Next column: 0,0,1,1,0,0,1,1... etc.
  3. Add intermediate columns for each gate — work left to right through the circuit.
  4. Complete the final output column.

Common Mistakes

  • Wrong number of rows — always 2ⁿ (2 inputs = 4 rows, 3 inputs = 8 rows)
  • Skipping intermediate columns — always add a column for each gate in a multi-gate circuit
  • Incorrect input ordering — use systematic binary counting (00, 01, 10, 11)
  • Confusing NOT(A AND B) with NOT A AND NOT B — these give different results
  • Applying NOT to entire expression when it only applies to one part
