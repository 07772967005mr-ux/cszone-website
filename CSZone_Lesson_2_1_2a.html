2.1.2a — Flowcharts
Component 2: Programming  ·  ~18 min

What is a Flowchart?

A flowchart is a visual diagram that represents an algorithm using standard symbols connected by arrows. Flowcharts are used to plan and communicate how a program will work before writing code.

Standard Flowchart Symbols

ShapeNamePurpose
Oval / Rounded rectangleTerminator (Start/Stop)Marks the beginning and end of a flowchart
RectangleProcessA processing step — e.g., calculation, assignment, output
ParallelogramInput/OutputData being input or output (display to screen)
DiamondDecisionA yes/no question — branches the flow based on a condition
ArrowFlow lineShows the direction of flow between steps
Memory trick: O = Oval (stOp/start), R = Rectangle (pRocess), P = Parallelogram (Put in/out), D = Diamond (Decision)

Control Structures in Flowcharts

Sequence

Steps follow one after another in order — boxes connected top to bottom with arrows.

Selection (branching)

A diamond shape with two exits: YES and NO. The flow branches based on whether a condition is true or false.

Example: Is score >= 50? → YES → "Pass" ; NO → "Fail"

Iteration (looping)

An arrow loops back to an earlier point, creating a loop. The loop continues until a decision condition is met. The decision (diamond) controls when the loop exits.

Example: Loop that keeps asking for a valid password until one is entered correctly.

Common Mistakes

  • Using a rectangle for decisions — MUST use a diamond
  • Using a diamond for processes — diamonds are ONLY for yes/no decisions
  • Forgetting a terminator (START/STOP oval) at the beginning and end
  • Arrows without labels on decision diamonds — YES and NO must be labelled
  • Confusing input/output (parallelogram) with process (rectangle)
